Sometimes, programmers must repeat the same part of code several times. Assume you go to work every day, Monday to Friday, and greet your colleague Nessie as you get to your office: "Hi, Nessie!" Let's simulate this situation using the tools we already have:

console.log("1, Hi, Nessie!"); // Monday
console.log("2, Hi, Nessie!"); // Tuesday
console.log("3, Hi, Nessie!"); // Wednesday
console.log("4, Hi, Nessie!"); // Thursday
console.log("5, Hi, Nessie!"); // Friday

We can create a loop instead of the repetition:

let i;
for (i = 1; i <= 5; i++) {
    console.log(i + ", Hi Nessie!");
}

This is way faster and more convenient, especially if we have to repeat the same stuff ten times, 100 times, or even more.

Let's zoom in on the syntax of the for loop. We set a variable before the loop: the i counter. Next, the condition inside the brackets has three parts: (...;...;...). The initial value of the counter variable comes first; in our case, it equals 1. The second part is the condition in which the loop keeps working, which, in our case, is i <= 5. Finally, in the third part, we indicate how exactly the counter should change from one iteration to another. The i++ recording is the same as i = i + 1, that is, the counter ranges from 1 to 5 with the step of 1 at each iteration. Any part of the condition inside the brackets is optional and can be omitted.

Infinite loop

An infinite loop is a loop whose execution never stops because there is no condition for exiting the loop (the second condition in the brackets). To write an infinite loop, skip the condition for exiting the loop to make a construction like this:

for (i=0; ; i++) {
    console.log(i);
}

Or this:

for (i=0; ;) {
    console.log(i);
}

Or even this:

for (; ;) {
    console.log('Hi!');
}

Why is that? Since none of the conditions in brackets are mandatory, we can skip a few or even all of them. But since we cut the condition for exiting the loop, the loop becomes infinite.

It's essential to exercise caution when using infinite loops in programming. As a result, your computer's memory will eventually become overloaded, leading to a system freeze. When this occurs, your only option is to reboot your computer in emergency mode. However, there are some situations where infinite loops are necessary â€” such as when programming games or microcontrollers. In these cases, using an infinite loop is a reasonable approach.

For...in loop

Let's examine another loop type related to the for loop category, the for...in loop:

let animals = ['cat', 'dog', 'turtle'];

for (i in animals) {
  console.log(animals[i]);
}

We get the following as the output:

cat
dog
turtle

In this example, we created an array containing the names of animals. Next, we go through the indexes of each animal in the array and display the names of the animals, one per line.

Note that the for...in loop passes only through the object's enumerable properties, that is, by properties that can be counted.

Conclusion

Loops are crucial in programming as they allow developers to repeat the same code multiple times without writing it repeatedly. They save time and make the code more organized and efficient. We have learned about two types of loops: the for loop and the for...in loop. We have also discussed the importance of having a condition to exit a loop and the potential dangers of infinite loops. By mastering loops, programmers can write more powerful and sophisticated programs.
